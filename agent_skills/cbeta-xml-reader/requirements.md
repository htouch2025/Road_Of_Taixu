# Requirements: CBETA XML Reader — 太虚大师全书

> 此文件记录纯用户要求。实现方法见 [SKILL.md](SKILL.md)。
> 最后更新：2026-06-20

## 触发条件

当需要读取、导航 CBETA TEI P5 XML 源文件（太虚大师全书）、提取文章目录与层级、交叉比对纸质版目录、或处理旧版 DOC/HTML 并对照 CBETA 权威版本时激活。

## 数据来源

R1. 所有 CBETA 读取操作均以本地 `_data/cbeta/TX/` 目录下的 40 个 TX XML 文件为唯一数据源。

R2. 绝不使用 buddha MCP（`mcp__buddha__*`）系列工具。

R3. 不得将完整 XML 文件加载入上下文。所有读取必须走「先查目录定位 → 再定点读取」路径。

## 语言与文字

R4. 本项目所有产出（目录树、JSON、备註等）统一采用繁体中文，与 CBETA 源文件一致，不做繁简转换。

R5. 用户搜索输入可能为简体，需先转换为繁体再匹配 CBETA 内容。

## 目录提取 — 编级

R6. 编级目录必须提取 level 1（子目类别）和 level 2（篇名）。

R7. 提取后必须执行两项复核：
    a. 检查是否有「（附）」前缀的 level-2 条目，若有应合并入前一篇文章，不得独立成篇。
    b. 确认编内文章编号跨子目连续递增，不复位。

R8. 编级目录必须同时产出 MD 和 JSON 两份文件。MD 中篇目的「子目」层须带数字编号。

## 目录提取 — 单篇文章

R9. 单篇文章目录树必须使用 shift rule 为每个 `<cb:mulu>` 条目分派语义标签（部/章/节/小节）。

R10. 必须过滤掉纲要、目次、科分、目錄、目録等纯结构预览条目。

R11. 必须保留「前言」条目。

R12. 每篇文章必须同时产出两份文件：
    a. MD（人读）：清洁目录树，不含 CBETA 内部标记。
    b. JSON（机读）：含 article、cbeta_id、编、子目、処理、tree（label/level/semantic/n/children）。

## MD 目录树格式

R13. 层级使用 `- ` 嵌套列表表示，每层缩进 4 个空格，不得使用树形绘制字符（├──、└──、│）。

R14. 篇名顶格；若有题注紧接一行；其后空一行再放 `## 目錄` 标题。

R15. 目录树末尾须附「備註」段，说明特殊处理（如移除纲要、保留前言等）。

## 题注处理

R16. 必须提取题下注释中的讲说时间、地点、记录者信息。

R17. 不得提取刊载来源（属于篇尾附注，非题下注释）。

R18. 纪年必须规范化：民国纪年 +1911 → 公元，宣统纪年 +1908 → 公元。

R19. 规范化后的日期格式：月份前保留空格（如「1930 年 1 月」），季节前不加空格（如「1915 年春」）。

R20. 原始 `——…——` 外围符号转为全角括号 `（…）`，日期与地点间加中文逗号。

## 标题编号空格规则

R21. 章节编号前缀与标题正文之间统一使用一个全角空格。

R22. 六种前缀模式均须正确处理：CJK 裸数字、第X章/节/篇、天干、天干+数字组合、全角数字、以及裸数字后紧跟另一数字的特殊情形（如「二三法印」→「二　三法印」）。

R22a. 裸数字前缀始终只取第一个数字字符，不贪婪匹配连续数字。例如：「一五蘊」→「一　五蘊」（非「一五　蘊」）；「二三法印」→「二　三法印」（非「二三　法印」）。

R23. 天干+数字复合前缀（如「甲一」）不得被拆开，须作为一个整体前缀。同样，「甲一證信」→「甲一　證信」（非「甲　　一證信」）。

## 篇末注释

R24. 篇末注释（back-notes）必须从 `<back>` 区域提取。

R25. 正文中须在对应位置插入纯数字标注（无链接），标题后的注释号与标题正文间空一格。

R26. 文末注释格式为 `- **N**：正文`。

R27. 篇末的记录者信息和刊载来源必须一并提取。

R28. 全文输出不得使用任何 Obsidian 特有语法（无 `[[...]]` 链接、无 `^` 块锚点、无 `{#...}` 自定义锚点），须为纯文本 Markdown。

## 全文输出格式

R29. 全文 Markdown 输出不得包含 `## 正文` 分隔标题。目录树结束后直接进入正文内容。

R30. 正文中不得出现 `### 前言` 标题行。前言原文直接作为正文首段输出，前方保留与目录树之间的视觉间距（源码中三行空行，渲染后合为一行）。

R31. 正文内的章节标题从 `##`（H2）起步。因 `## 正文` 已删除，原 H3→H2、H4→H3，依此类推整体上提一级。

R32. 篇末的记录者信息（如 `（碧松記）`）和刊载来源（如 `（原見海潮音月刊十八卷十一期）`）均须完整提取，不得遗漏其中任何一段。当两者嵌套在同一 XML 元素内时，须各自独立提取后拼接输出。

## 解经类文章

R33. 对于采用 orig/commentary div 结构的解经类文章，经论原文和讲义注解均须完整提取，不得遗漏正文段落。

R34. 经论原文（type="orig"）须在视觉上与讲义注解（type="commentary"）有所区分。

## 交叉校对

R35. CBETA 为权威版本。HTML/网页版目录为次要参考。纸质书总目录为最终仲裁者。

R36. 章名不得被误列为独立文章。

## 质量检验

R37. 每次产出后必须确认：层级分派正确、纲要已过滤、前言已保留、编号独立起编、附錄已合并、题注已规范化、正文无 `## 正文` 和 `### 前言` 标题行、正文标题从 H2 起步、篇末记录者与刊载来源均完整、目录与正文间保留三行空行、篇末注释已完整提取。

## 预排版文本块

R38. CBETA 中由文本字符组成的图表（如印度小乘分裂谱系图），在 XML 中以 `<p cb:type="pre">` 标记，须识别并特殊处理。

R39. 预排版块的每一行由 `<lb/>` 分隔；提取时 `<lb/>` 必须转换为换行符，前导全角空格（　）必须保留，不得压缩空白。

R40. 预排版块输出时须以 Markdown 代码围栏（```）包裹，以在渲染器中保持等宽对齐。


## 字数统计

R41. 提取正文时须统计全文（除「## 目錄」区块外）的中文字符数（CJK），并以万字为单位显示。

R42. 正文 MD 中，字数标注以 **X.X 万字** 格式置于 byline（或附录标题区）之后、## 目錄 之前，保留一位小数。

R43. 字数须回写入编目录 JSON（字数 字段，存原始 CJK 字符整数），并同步更新编目录 MD，以（X.X 万字）格式显示在标题后、题注前。

R44. regenerate_catalog_md() 中子目排列顺序须从 JSON 的 子目 数组读取而非按字母排序，以保持原始顺序。
## 图片提取

R45. CBETA XML 中的 `<figure><graphic url="..."/>` 元素须被识别并转换为本地 Markdown 图片引用。

R46. 图片引用中的相对路径必须动态计算（`os.path.relpath(figures_dir, output_dir)`），不得硬编码固定层级的 `../`，以适应输出目录的不同深度。

R47. 图片文件存放于 `_research/figures/TX/` 目录，从 cbeta-git/CBR2X-figures 仓库获取。
