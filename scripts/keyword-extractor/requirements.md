## 一、工程定位

这是太虚大师全书知识体系的一个子工程：为全书的每篇文章自动生成**关键词**，写入文章 YAML frontmatter 的 `keywords` 字段。最终目标是让读者可以通过关键词跨文章检索、发现概念关联、理解太虚思想体系的全貌。

## 二、关键词的定义

### 2.1 关键词包含两类

| 类型 | 定义 | 示例（以《佛理要略》为例） |
|------|------|------|
| **文中词** | 文章正文中明确出现和展开讨论的概念术语 | `因緣生法`、`三法印`、`五乘共法`、`涅槃寂靜` |
| **义理维度词** | 文章中没有直接出现这个词，但表达了该义理内涵、所属方法论立场、或对话传统 | `判教`、`佛法體系`、`通論`、`入門` |

两类关键词放在同一个 `keywords` 字段里，不分开。

### 2.2 关键词 vs. themes

| 字段 | 内容 | 当前状态 |
|------|------|:--:|
| `keywords` | 关键词列表（文中词 + 义理维度词，扁平列表） | **本期工程目标** |
| `themes` | 文章内容摘要（一段中文文字） | **暂不填充，后续处理** |

### 2.3 关键词的粒度

以**文章对该概念的实际讨论深度**为准：
- 如果文章系统讨论唯识三性 → 标 `三性`
- 如果文章泛泛涉及唯识整体 → 标 `唯識`
- 不标全文通用、无区分度的词（如 `佛法`、`佛教`）

### 2.4 关键词的层级

关键词是**扁平列表**，不分组、不嵌套。如：

```yaml
keywords:
  - 因緣生法
  - 三法印
  - 五乘共法
  - 唯識（法相唯識）
  - 判教
```

## 三、数量与篇幅的关系

对数式缩放，不线性增长：

| 文章字数（原始 CJK 字符） | 关键词数量 |
|------|:--:|
| ≤ 5,000 | 5–8 |
| 5,000–15,000 | 8–12 |
| 15,000–30,000 | 10–15 |
| > 30,000 | 12–18 |

不做章节级关键词，仅文章级。

## 四、术语一致性与双轨制

同一概念在不同文章中可能有不同叫法。处理规则：

- **双轨标注**：`標準詞（原文詞）`
  - 如：`唯識（法相唯識）`、`緣起（因緣所生法）`
- 若标准词与原文词相同，不写括号：`三法印`
- 标准词由**标准术语表**（`keyword_vocabulary.json`）定义，从全书实际内容中归纳

## 五、内容类型的自适应

全书 20 编（诗存除外）覆盖截然不同的内容类型。采用**统一框架 + 自适应调整**的策略：

| 藏 | 编 | 关键词侧重 |
|:--:|-----|------|
| 甲 法藏 | 1–7 佛法教理 | 教理概念、判教术语、宗派名 |
| 乙 制藏 | 8–10 律仪制度 | 戒条名、制度概念、僧制术语 |
| 丙 论藏 | 11–14 论说著作 | 论证方法、真现实论体系专有概念、论辩对象 |
| 丁 杂藏 | 15–19 时论等 | 社会概念、人物名、事件、机构、流派 |

LLM prompt 中根据编的类型给出不同的示例和侧重方向，但核心提取逻辑不变。

## 六、提取方法

**LLM 生成 + 人工校核**。

诗存（第二十编）暂不处理。

## 七、工程分两阶段

### 阶段一：标准术语表（`keyword_vocabulary.json`）

**目标**：从全书实际内容中归纳出太虚使用的核心术语体系，确立每个术语的标准形式。

**方法**：外部权威词典做种子 + 字符串匹配 + LLM 策展（详见 [术语表构建_资源与技术路径.md](术语表构建_资源与技术路径.md)）

- 从 DILA 佛教词典（丁福保、南山律學、翻譯名義大集、Soothill-Hodous）提取头术语作为种子词库
- 在太虚全书文本中做字符串匹配，获取每条术语的篇内频次
- 通过 CBETA API 查询全藏频次做通用词过滤
- LLM 审阅术语列表 → 归并异名、确立标准形式 → 分类
- 人工校核定稿

**产出**：`_research/keyword_vocabulary.json`

### 阶段二：关键词提取脚本（`extract_keywords.py`）

**目标**：为每篇文章自动生成关键词，写入 frontmatter。

**使用场景**：逐篇增量提取，嵌入正文提取流程。1,368 篇文章的提取正文仍在陆续进行中，关键词提取将伴随提取进度推进——每完成一篇（或几篇）正文的提取，即可对该篇运行关键词提取。不做一次性全量批跑。

**方法**：
- 脚本读取单篇文章 MD 全文（排除 `## 目錄` 区块）
- 发送给 LLM，LLM 参考标准术语表 + 该文章的候选术语（Phase A 匹配结果）生成关键词
- 首次运行时不强制约束词汇表（LLM 自由判断标准词）
- 后续轮次可逐步收紧
- 亦支持 `--batch` 模式，方便对整批文章集中提取
- 双轨标注：`標準詞（原文詞）`

**产出**：每篇文章的 YAML frontmatter `keywords` 字段被逐篇填充。


## 七点五、义理维度词来源：太虚研究文献子工程

义理维度词（如 `判教`、`通論`、`入門`、`人間佛教` 等）是词典匹配无法产出的——它们反映的是对太虚思想的概括性判断和方法论立场。学术论文天然以概括和定位为核心，标题、摘要、关键词中大量出现此类术语。

因此设立一个独立子工程，从太虚研究文献中系统提取义理维度词候选集，汇入阶段一 Phase B 的 LLM 策展环节。

**来源路径**：
```
_research/taixu_papers
```

**与主工程的关系**：该子工程产出的「太虚义理维度术语候选集」在阶段一 Phase B（LLM 策展）时与词典术语联合处理，不设独立流程。

**详细方案**：见 [术语表构建_资源与技术路径.md §3.6](术语表构建_资源与技术路径.md)。

**状态**：✅ 已完成（2026-06-24）。详见 [术语表构建_资源与技术路径.md §3.6](术语表构建_资源与技术路径.md) 和 `_research/taixu_dimensional_terms.json`。

## 八、不做什么

| 项目          | 说明                                          |
| ----------- | ------------------------------------------- |
| `themes` 字段 | 暂不填充                                        |
| 诗存（第二十编）    | 诗词的关键词体系需单独设计，暂不纳入                          |
| 章节级关键词      | 仅文章级                                        |
| 纯规则算法       | 算法仅用于收集统计证据（字符串匹配、频次查询），关键词的选取和归并决策权始终在 LLM |
| 预编标准词汇表     | 不从外部灌输，从全书内容归纳                              |
