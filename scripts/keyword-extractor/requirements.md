## 一、工程定位

这是太虚大师全书知识体系的一个子工程：为全书的每篇文章自动生成**四维标注**（要义 / 论域 / 论用 / 旨归），写入文章 YAML frontmatter 的 `concepts`、`domains`、`functions`、`bearings` 四个字段。最终目标是让读者可以通过多维度标签跨文章检索、发现概念关联、理解太虚思想体系的全貌。

## 二、四维标注体系

### 2.1 四个维度

| 维度 | 字段 | 定义 | 示例（以《佛理要略》为例） |
|------|------|------|------|
| 要义 | `concepts` | 文章重点展开的具体概念术语 | `因緣生法`、`三法印`、`五乘共法`、`涅槃寂靜` |
| 论域 | `domains` | 文章所属佛学领域 | `基礎教義`、`判教`、`佛法體系` |
| 论用 | `functions` | 写作意图与方法论立场 | `入門導引`、`綱要约論` |
| 旨归 | `bearings` | 终极关怀与对话传统 | `解脫道`、`菩薩道` |

四维各自独立，分别写入 frontmatter 的对应字段。无 `keywords` 字段。

### 2.2 `themes` 字段

`themes` 字段暂不填充，后续处理（见 §八）。

### 2.3 要义（concepts）的粒度

以文章对该概念的实际讨论深度为准：

- 如果文章系统讨论唯识三性 → 标 `三性`
- 如果文章泛泛涉及唯识整体 → 标 `唯識`
- 不标全文通用、无区分度的词（如 `佛法`、`佛教`）

此粒度规则仅适用于 `concepts` 维度。其他三维使用标准化值域（见 §十）。

### 2.4 扁平结构

四个维度各自为扁平列表，不分组、不嵌套。如：

```yaml
concepts:
  - 因緣生法
  - 三法印
  - 五乘共法
  - 唯識
domains:
  - 基礎教義
functions:
  - 入門導引
bearings:
  - 解脫道
```

## 三、数量与篇幅的关系

对数式缩放，不线性增长：

| 文章字数（原始 CJK 字符） | `concepts` 数量 | `domains` | `functions` | `bearings` |
|------|:--:|:--:|:--:|:--:|
| ≤ 5,000 | 5–8 | 1–3 | 1–3 | 1–3 |
| 5,000–15,000 | 8–12 | 1–3 | 1–3 | 1–3 |
| 15,000–30,000 | 10–15 | 1–3 | 1–3 | 1–3 |
| > 30,000 | 12–18 | 1–3 | 1–3 | 1–3 |

仅 `concepts` 按字数对数缩放；`domains`、`functions`、`bearings` 各维固定 1–3 个。
不做章节级关键词，仅文章级。

## 四、术语一致性与双轨制

同一概念在不同文章中可能有不同叫法。处理规则：

- **双轨标注**：`標準詞（原文詞）`
  - 如：`唯識（法相唯識）`、`緣起（因緣所生法）`
- 若标准词与原文词相同，不写括号：`三法印`
- 标准词由**标准术语表**（`keyword_vocabulary.json`）定义，从全书实际内容中归纳

此规则仅适用于 `concepts` 维度。其他三维的值域来自标准化标签（`labels.json`），不涉及原文词变体问题。

## 五、内容类型的自适应

全书 20 编（诗存除外）覆盖截然不同的内容类型。采用**统一框架 + 自适应调整**的策略：

| 藏 | 编 | 四维标注侧重 |
|:--:|-----|------|
| 甲 法藏 | 1–7 佛法教理 | 教理概念、判教术语、宗派名 |
| 乙 制藏 | 8–10 律仪制度 | 戒条名、制度概念、僧制术语 |
| 丙 论藏 | 11–14 论说著作 | 论证方法、真现实论体系专有概念、论辩对象 |
| 丁 杂藏 | 15–19 时论等 | 社会概念、人物名、事件、机构、流派 |

LLM prompt 中根据编的类型给出不同的示例和侧重方向，但核心提取逻辑不变。

## 六、提取方法

**LLM 生成 + 人工校核**。

诗存（第二十编）暂不处理。

## 七、工程分两阶段

### 阶段一：标准术语表（`keyword_vocabulary.json`）

**状态**：✅ 已完成（2026-06-24），词表 v4（1,392 条，8 域 92 子域组合）。实施记录见 [references/keyword_vocabulary_task.md](references/keyword_vocabulary_task.md)

**目标**：从全书实际内容中归纳出太虚使用的核心术语体系，确立每个术语的标准形式。

**方法**：外部权威词典做种子 + 字符串匹配 + LLM 策展（详见 [术语表构建_资源与技术路径.md](术语表构建_资源与技术路径.md)）

- 从 DILA 佛教词典（丁福保、南山律學、翻譯名義大集、Soothill-Hodous）提取头术语作为种子词库
- 在太虚全书文本中做字符串匹配，获取每条术语的篇内频次
- 通过 CBETA API 查询全藏频次做通用词过滤
- LLM 审阅术语列表 → 归并异名、确立标准形式 → 分类
- 人工校核定稿

**产出**：`vocabulary/keyword_vocabulary_v4.json`

### 阶段二：关键词提取脚本

**状态**：🔄 进行中（2026-06-24 起，已处理 38 篇）

**目标**：为每篇文章自动生成关键词，写入 frontmatter。

**当前方法**（v2，四维标注）：Phase A（程序化词表匹配 → 候选 JSON）+ Phase B（LLM 四维精判），详见 [SKILL.md](SKILL.md) §一。

**产出**：每篇文章的 YAML frontmatter 逐篇写入 `concepts` / `domains` / `functions` / `bearings` 四个字段。


## 七点五、义理维度词来源：太虚研究文献子工程

义理维度词（如 `判教`、`通論`、`入門`、`人間佛教` 等）是词典匹配无法产出的——它们反映的是对太虚思想的概括性判断和方法论立场。学术论文天然以概括和定位为核心，标题、摘要、关键词中大量出现此类术语。

因此设立一个独立子工程，从太虚研究文献中系统提取义理维度词候选集，汇入阶段一 Phase B 的 LLM 策展环节。

**来源路径**：
```
_research/taixu_papers
```

**与主工程的关系**：该子工程产出的「太虚义理维度术语候选集」在阶段一 Phase B（LLM 策展）时与词典术语联合处理，不设独立流程。

**详细方案**：见 [术语表构建_资源与技术路径.md §3.6](术语表构建_资源与技术路径.md)。

**状态**：✅ 已完成（2026-06-24）。详见 [术语表构建_资源与技术路径.md §3.6](术语表构建_资源与技术路径.md) 和 `_research/taixu_dimensional_terms.json`。

## 十一、缺口词记录与自动补入

### 11.1 需求

Phase B 的 LLM 标注可能产出不在标准术语表中的新概念。需要一套自动机制：
- 记录每次 `--apply` 产生的缺口词
- 定期审查缺口词频次
- 频次 ≥2 的缺口词自动补入标准术语表
- 整个过程无需人工干预

### 11.2 记录机制

`--apply` 写入 frontmatter 时，将标注值与当前术语表做对比，不在表中的条目自动记入 `logs/_keyword_gap_log.jsonl`（按文章去重）。

### 11.3 触发阈值与自动补入

`logs/_kw_counter.json` 跟踪已处理文章数（按文章路径去重）。每满 5 篇自动触发以下审查流程：

1. **去假阳性**：每个缺口词逐一查当前词表（标准词 + variants，经简繁标准化），已在表中的直接剔除
2. **concepts 频次 ≥2** → 自动补入词表，通过同文章其他已存术语推断 domain/subdomain
3. **domains / functions / bearings 频次 ≥2** → 自动补入 `vocabulary/labels.json`
4. **频次 =1** → 保留在日志中，等待后续文章累积跨越阈值
5. **回写日志** → 只保留真实缺口条目，假阳性条目自动移除

### 11.4 手动入口

保留两个手动 flag 用于中途审查：

```bash
python3 scripts/extract_keywords.py --review-gaps        # 只读报告
python3 scripts/extract_keywords.py --review-gaps-apply  # 审查 + 补入
```
