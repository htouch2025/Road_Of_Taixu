## 一、任务背景

这是太虚大师全书关键词提取工程的第一步。

整体工程目标：为全书每篇文章的 YAML frontmatter 自动填充 `keywords` 字段（关键词列表），由 LLM 读取全文后生成，人工校核。

但在逐篇生成关键词之前，需要先有一套**标准术语表**（standard vocabulary），作为关键词双轨标注的依据。原因是同一概念在太虚不同文章里可能用不同的词（如「唯識」「法相唯識」「瑜伽行派」），标准术语表统一标准形式，确保跨文章一致。

标准术语表不预先凭空编造，而应该从全书实际内容中归纳生长出来。

## 三、任务目标

产出文件：`_research/keyword_vocabulary.json`

这是一个标准术语参考表，格式如下：

```json
{
  "meta": {
    "created": "2026-06-22",
    "source": "从太虚大师全书 19 编 1,358 篇文章中提炼",
    "total_terms": 150
  },
  "terms": [
    {
      "standard": "唯識",
      "variants": ["法相唯識", "瑜伽行派"],
      "domain": "教理",
      "subdomain": "大乘有宗",
      "source_bian": [1, 4, 6],
      "frequency": 47,
      "note": ""
    },
    {
      "standard": "因緣生法",
      "variants": ["因緣所生法", "緣起"],
      "domain": "教理",
      "subdomain": "基礎教義",
      "source_bian": [1, 2, 3, 4, 5, 6, 7],
      "frequency": 89,
      "note": ""
    }
  ]
}
```

字段说明：

| 字段 | 类型 | 说明 |
|------|------|------|
| `standard` | string | 标准形式（繁体中文） |
| `variants` | string[] | 太虚在不同文章中使用的同义变体 |
| `domain` | string | 领域大类：教理 / 行持 / 历史 / 制度 / 社会 / 其他 |
| `subdomain` | string | 细分领域，如「大乘有宗」「戒律」「禪修」等 |
| `source_bian` | int[] | 术语出现的编编号列表 |
| `frequency` | int | 在全书文章中出现的频次（基于实际统计） |
| `note` | string | 备註说明 |

### 术语范围

标准术语表应覆盖：
- **教理术语**：五乘共法、三乘共法、大乘法、因緣生法、三法印、諸行無常、諸法無我、涅槃寂靜、唯識、中觀、如來藏、真如、空性、般若、阿賴耶識、末那識、三性三無性、轉依、菩提心、六度、四攝、等等——以及它们在太虚行文中出现的各种变体形式
- **行持术语**：禪修、念佛、持戒、止觀、四禪八定、六度萬行 等
- **历史术语**：中國佛教、印度佛教、藏傳佛教、南傳佛教、宗派源流、佛教改革 等
- **制度术语**：戒律、僧制、羯磨、布薩、安居 等
- **太虚特有概念**：真现实論、法界圓覺、判攝、佛教革命、人間佛教、僧教育 等
- **人名/地名**：高频出现的佛教人物和地点（如馬鳴、龍樹、無著、世親 等）

### 排除项

- 全书通用词（如「佛法」「佛教」「佛陀」除非有特殊限定）
- 过于琐碎的表述（如出现在一处且无体系价值的短语）
- 诗存（后续单独处理）

## 四、提炼方法：三层推进

由于全书 405 万字无法一次性读入 LLM 上下文，采用三层分步提炼：

### 第一层：编目录 + 文章 TOC 树（结构信息全覆盖）

**输入**：19 份编目录 JSON + 每篇文章 MD 中的 `## 目錄` 区块

**方法**：
1. 从编目录 JSON 中提取所有文章标题（1,358 条）+ 编名 + 子目名
2. 从每篇文章 MD 的前 200 行提取 `## 目錄` 区块（轻量，不读全文）
3. 将标题+编名+子目名+TOC 汇总，发给 LLM，让 LLM 从中归纳出：
   - 出现了哪些核心佛教概念
   - 太虚的概念分类框架（编名和子目名本身就是他体系的体现）
   - 同一概念在不同文章标题中的表述变体

**产出**：第一版术语清单（侧重于文章标题和结构层面反映的概念）

### 第二层：法藏 7 编逐篇全量（教理核心）

**输入**：法藏 7 编（01-07）的 202 篇文章**全文**

**方法**：
1. 写一个脚本（可用 Python + LLM API），逐篇读取文章全文
2. 对每篇文章，让 LLM 提取该文讨论的**核心佛教术语**及其在文中的**具体用词**
3. 重点记录：术语的标准形式 + 文中使用的具体词形 + 所属的论域
4. 将所有输出汇总，与第一层结果合并

**注意**：
- 法藏 202 篇文章约 202 万字，需分批处理
- 每篇文章的 LLM prompt 应简洁：只需返回术语列表，不需要摘要
- 优先处理字数多、结构复杂的文章（这些术语密度最高）

**产出**：第二版术语清单（覆盖教理核心区）

### 第三层：制藏/论藏/杂藏抽样补充（查漏补缺）

**输入**：制藏（08-10）、论藏（11-14）、杂藏（15-19）各子目代表性文章

**方法**：
1. 每个子目选取最长的 1-2 篇（因为它们内容最充分），约 60-80 篇
2. 逐篇 LLM 提取术语
3. 与第二层结果对比，找出法藏中未出现但在其他藏中出现的**特有术语**
4. 特别关注：
   - 制藏：律仪术语、戒条名称、僧制概念
   - 论藏：真现实论体系专有概念
   - 杂藏：时论中涉及的社会/政治概念、书评术语、书信中的人名/地名

**产出**：第三版术语清单（覆盖全面）

### 汇总归一化

将三层结果合并：
1. **去重**：同一概念的不同词形 → 归并到标准形式
2. **确立标准形式**：选取最规范/最常用的词形为标准
3. **分类**：按 domain（教理/行持/历史/制度/社会/其他）粗分 + subdomain 细分
4. **统计频次**：记录每个术语在多少篇文章中出现
5. **输出最终 JSON**：写入 `_research/keyword_vocabulary.json`

## 五、产出物

1. **`_research/keyword_vocabulary.json`** — 标准术语表（核心产出）
2. **`_research/keyword_vocabulary.md`** — 供人阅读的术语表（可选，带分类和说明）

## 六、术语表的使用方式

标准术语表产成后，在后续关键词提取（`extract_keywords.py`）中作为**参考但不强制约束**：

- LLM 对每篇文章提取关键词时，参考词汇表确保术语标准化
- 关键词采用双轨格式：`標準詞（原文詞）`，如 `唯識（法相唯識）`
- 若某文章出现词汇表中未收录的新术语，将其加入词汇表（持续更新）
