# 太虛文章關鍵詞提取方法

> 最後更新：2026-06-24
> 狀態：已驗證（雙樣本：佛學概論、佛理要略）— 改進中
> 詞表版本：keyword_vocabulary_v4.json（1371 條，含維度詞）

## 一、总体流程

```
文章 MD
   │
   ▼
┌──────────────────────────────┐
│ Phase A：程序化粗筛（零成本）    │
│ extract_keywords.py --article │
│ → 候选 JSON（~400 条→去重）     │
└──────────────────────────────┘
   │
   ▼
┌──────────────────────────────┐
│ Phase B：LLM 精判（Codex）     │
│ 1. 读文章全文                  │
│ 2. 审候选集，区分：             │
│    · 核心讨论 — 提名            │
│    · 顺带提及 — 跳过            │
│    · 字面误中 — 跳过            │
│ 3. 补充自由关键词（≤5 个）      │
│ 4. 双轨标注：标准词（原文词）    │
│ 5. 按文章长度控制总量（5-18）    │
└──────────────────────────────┘
   │
   ▼
┌──────────────────────────────┐
│ 写入 frontmatter              │
│ extract_keywords.py --apply   │
└──────────────────────────────┘
```

## 二、工具链

| 工具 | 路径 | 功能 |
|------|------|------|
| 候选集生成 | `scripts/extract_keywords.py --article` | Phase A 匹配 + 输出候选 JSON |
| 关键词写入 | `scripts/extract_keywords.py --apply` | 写回 YAML frontmatter |
| 批次模式 | `scripts/extract_keywords.py --catalog` | 批量处理编内文章 |

**依赖数据：**
- 術語表（含義理維度詞）：`vocabulary/keyword_vocabulary_v4.json`（1371 條，含異名與維度詞）
  合併來源：v3_draft + batch_01 + Phase2 全文匹配 + Phase3 抽樣 + dimensional terms

## 三、Phase B 判断标准

### 3.1 候选筛选三分类

| 判定 | 标准 | 动作 |
|------|------|------|
| **核心讨论** | 文章有专门段落/章节展开此概念 | → 提名关键词 |
| **顺带提及** | 文中出现但非论述焦点 | → 跳过 |
| **字面误中** | 子串匹配、图表枚举项、大纲标签 | → 跳过 |

### 3.2 自由关键词

以下情况使用自由关键词（术语表外）：
- 术语表尚未收录的重要概念（如「佛學」「結集」）
- 文章的功能类型描述（如「入門」「通論」「專論」）
- 义理维度词（如「判教」「佛法體系」）

每篇文章自由關鍵詞不超過 5 個。
自由關鍵詞在 `--apply` 寫入時自動記錄到 `logs/_keyword_gap_log.jsonl`，
供後續批量策展術語表使用。

### 3.5 候選排序解讀（Phase B 審閱指引）

Phase A 候選集按加權分排序：`score = occurrences + 3 × 標題命中數`。
- 出現在章節標題中的術語獲得 +3 加權，排序優先於僅在正文高頻出現的低信號術語
- Phase B 審閱時優先檢查排序靠前（高分）的候選——這些最有可能是核心討論概念
- 排序靠後的低頻、無標題命中的候選多為順帶提及或字面誤中，可快速跳過
- 候選 JSON 頂部的 `summary` 物件提供術語密度快覽：候選總數、單次出現佔比、高頻術語列表

### 3.3 双轨标注规则

- 标准词与原文一致 → 直接写：`三法印`
- 标准词与原文不同 → 双轨：`緣起（因緣）`
- 标准词可变体映射 → 双轨：`三藏（經律論）`
- 自由关键词 → 无括号：`入門`

### 3.4 数量控制

| 文章字数（CJK） | 关键词数 |
|:--:|:--:|
| ≤ 5,000 | 5–8 |
| 5,000–15,000 | 8–12 |
| 15,000–30,000 | 10–15 |
| > 30,000 | 12–18 |

## 四、注意事项

1. **只标文章级关键词**，不做章节级。
2. **排除全文通用无区分度的词**（如「佛法」「佛教」）。
3. **诗存（第二十编）暂不处理**。
4. **按编推进**更合理——同一编文章类型接近，判断标准一致。
5. **发现术语表缺口时记录**，但不阻塞单篇文章的提取。

### 待驗證（跨編多文體，v4 詞表 + 標題加權排序）

以上雙樣本來自同一編同一文體（編01 概論·通論），尚不足證明方法泛用性。
下文列出跨編驗證計劃，覆蓋三種文體各至少一篇：

| 樣本 | 編 | 文體 | 預期特點 |
|------|----|------|----------|
| 真現實論（選篇） | 第13編 真現實論宗用論 | 論藏·理論密度高 | 候選術語密集，需更嚴格區分核心討論與順帶提及 |
| 講演（選代表性講演） | 第18編 講演 | 口語化講演 | 自由關鍵詞比例可能更高，候選命中模式與書面文章不同 |
| 酬對（選代表性對話） | 第17編 酬對 | 對話體酬答 | 短段落、快節奏對話，關鍵詞稀疏，需更多自由關鍵詞補充 |

#### 跨編驗證指標

- 各篇候選總數變化
- 前 10 候選中標題命中術語佔比
- 單次出現候選佔比（應與優化前持平或略降）
- 自由關鍵詞數量及類型
- 三法印、一實相印等 v4 新增術語是否在候選中出現

#### 標題加權係數校準

標題命中加權係數初始值為 3。跨編驗證時觀察：
- 加權後排序是否合理（核心概念是否被推到前列）
- 如係數過大導致標題術語壓過真正高頻核心術語，調降至 2
- 如係數過小導致排序無明顯改善，調升至 4–5

## 六、質量閉環

### 6.1 術語表缺口日誌

`--apply` 寫入關鍵詞時，系統會自動將術語表中不存在的自由關鍵詞追加寫入 `logs/_keyword_gap_log.jsonl`。
每行一條 JSON：

```json
{"article_path": "...", "gap_terms": ["三法印", "一實相印"], "timestamp": "2026-06-24T..."}
```

### 6.2 策展流程

1. 每隔一編完成或累積約 20 篇文章後，檢視 `_keyword_gap_log.jsonl`
2. 篩選出現頻率 ≥ 2 篇的自由關鍵詞，判斷是否補入詞表
3. 補入詞表時更新 `keyword_vocabulary_v4.json`（增補 terms、更新 meta、重建 variant 索引）
4. 完成後可考慮升級版本號至 v5

## 七、使用示例

```bash
# 单篇文章
python scripts/extract_keywords.py --article _research/01_佛法總學/01_概論/01_佛學概論.md

# 写入关键词
python scripts/extract_keywords.py --apply _research/01_佛法總學/01_概論/01_佛學概論.md \
  --keywords '["佛學","教理行果","三藏（經律論）","結集","釋迦牟尼","小乘","大乘","緣起（因緣）","五乘","三法印","一實相印","入門"]'

# 佛理要略
python scripts/extract_keywords.py --apply _research/01_佛法總學/01_概論/02_佛理要略.md \
  --keywords '["因緣生","五乘","十二緣起","三法印","諸行無常","諸法無我","大乘","菩提心","總持","向上增進心","出離流轉心","普度成佛心"]'

# 批次：第一编前5篇
python scripts/extract_keywords.py \
  --catalog _research/01_佛法總學/_01_佛法總學_編目錄.json --start 1 --end 5
```
